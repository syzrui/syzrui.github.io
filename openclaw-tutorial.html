01

OpenClaw 是什么？

OpenClaw 是一个开源的、可以自己托管在电脑上的 AI 助手网关（Agent Gateway）。简单来说，它就像一座桥梁——将你选择的大语言模型（如 Claude、GPT-4、本地 Llama 等）与你日常使用的聊天软件（Telegram、WhatsApp、微信等）连接起来，让你可以随时随地通过熟悉的 App 与 AI 对话，并让 AI 帮你完成各种任务。

与普通的 ChatGPT 网页不同，OpenClaw 是"永远在线"的——它作为一个后台程序持续运行在你的电脑或服务器上，可以主动给你发消息、执行定时任务、管理文件，甚至自动回复邮件。

🌐

连接消息平台

通过 Telegram、WhatsApp、Discord、Slack 等 App 与 AI 对话

🤖

支持多种 AI 模型

云端模型（Claude、GPT）或本地模型（Ollama + Llama）均可

⚙️

自动执行任务

定时提醒、文件管理、网页浏览、邮件处理全部支持

🔒

数据本地化

运行在自己的机器上，敏感数据不上传至第三方服务器

💡
类比理解：把 OpenClaw 想象成你自己雇用的私人 AI 管家——住在你家（你的电脑）里，通过你指定的方式（Telegram/WhatsApp）联系你，帮你处理各种日常事务。
02

核心功能介绍

OpenClaw 的架构分为两部分："大脑"（Brain，负责调用 AI 模型和逻辑协调）和"双手"（Hands，负责执行具体操作如运行命令、管理文件）。以下是主要功能：

①

🗨️ 多平台聊天接入

支持 Telegram、WhatsApp、Discord、Slack、iMessage、Signal、微信 LINE、Matrix 等 20+ 个消息平台，用你最习惯的 App 与 AI 对话。

②

🔁 永久后台运行（Daemon 模式）

安装为系统服务（Linux 的 systemd / macOS 的 launchd），开机自启，电脑重启后无需手动重新运行，AI 助手始终在线。

③

🛠️ 技能（Skills）插件系统

可以安装各种技能插件，例如：网页浏览与搜索、Gmail 邮件管理、Google Calendar 日历提醒、文件读写、Shell 命令执行等。

④

⏰ 定时任务与主动提醒

可以设定 Cron 定时任务，让 AI 在特定时间主动给你发消息，比如每天早上总结新闻、提醒日程等。

⑤

🖥️ 浏览器控制

内置专用 Chrome/Chromium 浏览器，AI 可以自主浏览网页、截图、填写表单，完成复杂的网页自动化任务。

安装前准备

在开始安装之前，请确认你的电脑满足以下基本要求：

项目 要求 说明 必要性
操作系统 Windows 10/11、macOS 12+、Linux（Ubuntu 推荐） Windows 需要额外安装 WSL2 必须
Node.js 版本 22 或更高 安装脚本会自动检测并安装 必须
内存 4 GB RAM 以上 若运行本地模型则需要更多 必须
硬盘空间 至少 2 GB 可用空间 本地模型需要额外空间 必须
网络连接 稳定的互联网连接 用于下载依赖和调用 AI API 必须
AI API 密钥 Claude / OpenAI 等云端服务的密钥 本地模型（Ollama）可跳过此项 可选
Ollama 本地运行 AI 模型的工具 不想付费使用 API 时的替代方案 可选
⚠️
Windows 用户注意：OpenClaw 官方强烈建议在 Windows 上通过 WSL2（适用于 Linux 的 Windows 子系统）来运行，直接在 PowerShell 中安装可能不稳定。本教程将详细说明如何在 Windows 上配置 WSL2。
04

Windows 安装教程

Windows 安装分为两个阶段：先安装 WSL2（Linux 子系统），再安装 OpenClaw。

✓满足系统要求
1安装 WSL2
2安装 OpenClaw
3初始化配置
4连接平台
1

以管理员身份打开 PowerShell

点击 Windows 开始菜单，搜索 PowerShell，右键点击结果，选择"以管理员身份运行"。

2

安装 WSL2 和 Ubuntu

在管理员 PowerShell 窗口中输入以下命令并按 Enter：

PowerShell 
PS> wsl --install -d Ubuntu

安装过程大约需要 5~15 分钟，取决于网速。安装完成后，必须重启电脑。

ℹ️
重启后，Ubuntu 会自动打开并要求你设置一个 Linux 用户名和密码，这个密码只用于 WSL2 内部，与 Windows 账户无关，设置一个你记得住的密码即可。
3

在 WSL2 Ubuntu 中安装 OpenClaw

打开 Ubuntu 终端（在开始菜单搜索"Ubuntu"），然后输入以下一行命令：

bash (Ubuntu / WSL2) 
$ curl -fsSL https://openclaw.ai/install.sh | bash

这条命令会自动完成：检测 Node.js 版本 → 安装缺失的依赖 → 下载 OpenClaw → 启动初始化向导。

4

阅读并接受安全提示

安装过程中会出现一个安全警告界面，内容是说明 OpenClaw 具有执行命令和访问文件的高权限。请仔细阅读后，用方向键选择 Yes 并按 Enter 继续。

🔐
请认真阅读安全说明！OpenClaw 默认有权限执行 Shell 命令和访问文件，请在了解风险后再继续。
05

macOS / Linux 安装教程

macOS 和 Linux 的安装流程几乎完全相同，是最简单的安装路径，推荐大多数非 Windows 用户使用。

1

打开终端

macOS：使用 Spotlight（Command + Space），搜索"终端"或"Terminal"打开。
Linux：使用快捷键 Ctrl+Alt+T，或在应用菜单中找到"终端"。

2

运行安装命令

复制并粘贴以下命令到终端，按 Enter 执行：

bash 
$ curl -fsSL https://openclaw.ai/install.sh | bash

安装脚本会自动检测 Node.js 并在缺失时安装，随后启动 OpenClaw 初始化向导。等待安装完成（通常 2~5 分钟）。

3

（可选）如果你想先检查脚本内容

对于注重安全的用户，可以先下载脚本查看内容再执行：

bash 
$ curl -O https://openclaw.ai/install.sh
$ cat install.sh        # 查看脚本内容
$ bash install.sh       # 确认安全后再执行
4

（可选）通过 npm 手动安装

如果已经安装了 Node.js 22+，也可以用 npm 直接安装：

bash 
$ npm install -g openclaw@latest
$ openclaw onboard --install-daemon   # 启动向导并注册为后台服务
✅
macOS 用户福利：macOS 对 OpenClaw 的集成最为原生，支持 Apple Notes、Reminders（提醒事项）和 iMessage 等系统应用的直接集成。如果你使用 Mac Mini（特别是 M4），它已成为运行 OpenClaw 的首选硬件。
06

初始化向导（Onboard）

安装完成后，OpenClaw 会自动进入初始化向导（或你可以手动运行 openclaw onboard）。向导会引导你完成所有必要配置，以下是每个步骤的说明：

1

选择安装模式：快速启动 or 手动配置

向导第一步会问你选择 Quickstart（快速启动） 还是 Manual（手动配置）。

新手强烈推荐选择 Quickstart，它会自动为你选择合理的默认设置，后续也可以随时修改。

2

配置 AI 模型提供商

选择你想使用的 AI 模型来源：

选项说明
  ▸ Anthropic (Claude)     — 需要 Anthropic API 密钥
  ▸ OpenAI (GPT-4)        — 需要 OpenAI API 密钥  
  ▸ Google (Gemini)       — 需要 Google AI API 密钥
  ▸ Local / Ollama        — 本地运行，无需 API 密钥（免费）
  ▸ Skip for now          — 跳过，稍后配置
💰
没有 API 密钥？选择底部的 Skip for now 跳过，之后可以安装 Ollama 在本地免费运行 AI 模型。
3

选择默认 AI 模型

如果你选择了云端 AI 提供商，下一步会让你选择具体使用哪个模型。建议选择性价比较高的中端模型（如 claude-sonnet 或 gpt-4o-mini），而非最顶级模型，否则 API 费用较高。

4

选择技能插件（Skills）

向导会询问是否现在配置技能插件。新手推荐先选"No"跳过，等熟悉基本使用后再按需添加。技能包括网页搜索、邮件管理、日历等功能。

后续添加技能的命令 
$ openclaw skills install    # 查看并安装技能
5

安装 Hooks（钩子）

向导建议安装 3 个 Hooks，它们可以让 AI 更好地理解上下文和执行任务。推荐全部选择安装，但如果你不希望数据被本地记录，可以选择跳过。

6

完成安装，获取控制面板地址

向导完成后，终端会显示一个本地 URL 地址（类似 http://localhost:18789）和访问令牌（Access Token）。

🎉
请务必保存此地址和令牌！这是访问 OpenClaw 控制面板的凭证，类似于密码，丢失后需要重新生成。
07

连接消息平台

现在让 OpenClaw 与你常用的聊天 App 连接。以下以最常用的 Telegram 为例演示，WhatsApp 操作类似。

1

在 Telegram 中联系 BotFather

打开 Telegram，在搜索栏中搜索 @BotFather，点击并打开对话。

2

创建新 Bot

在 BotFather 对话中发送命令 /newbot，然后按提示输入你的机器人名称（随便起一个）。BotFather 会返回一串 API Token（格式类似 123456789:ABC-DEF...）。

3

将 Token 粘贴到 OpenClaw

在 OpenClaw 向导中选择 Telegram 作为频道，将上面复制的 Token 粘贴进去，按 Enter 确认。

bash 
$ openclaw channel add telegram
  Enter your Telegram Bot Token: 123456789:ABC-DEF...
4

开始对话！

在 Telegram 中搜索你刚创建的机器人用户名，点击 Start，然后向它发送任何消息，你的 OpenClaw AI 助手就会回应你了！

1

在 OpenClaw 向导中选择 WhatsApp

选择"显示二维码（QR Code）"选项，终端会显示一个二维码图案。

2

在手机上扫描二维码

打开手机上的 WhatsApp，进入"设置"→ 点击右上角的 "二维码图标"→ 选择"扫描码"，扫描终端中显示的二维码。

3

完成配对

扫描成功后，如果提示输入手机号码，输入你的 WhatsApp 注册手机号即可完成配对。之后直接向自己的 WhatsApp 账户发消息就能与 AI 对话。

08

使用控制面板

OpenClaw 提供了一个基于浏览器的控制面板，用于查看状态、管理配置和调试问题。

1

打开控制面板

在终端运行以下命令，会自动打开浏览器：

bash 
$ openclaw dashboard

或者直接在浏览器地址栏输入安装时提供的地址（如 http://localhost:18789）并输入访问令牌登录。

2

常用管理命令

bash — 常用命令速查 
$ openclaw status          # 查看网关运行状态
$ openclaw doctor          # 自动检查配置问题
$ openclaw logs --follow   # 实时查看运行日志
$ openclaw gateway restart # 重启网关服务
$ npm update -g openclaw   # 更新到最新版本
09

重要安全须知

🚨
OpenClaw 是一个高权限工具！它可以在你的系统上执行命令、读写文件、访问网络。请认真阅读以下安全建议。
1

🔑 保护好你的访问令牌和配置文件

OpenClaw 的配置文件（位于 ~/.openclaw/）包含你的 API 密钥和访问令牌，权限等同于密码。不要将这个目录分享给任何人，也不要上传到 GitHub 等公开平台。

2

🌐 不要将控制面板暴露在公网

默认情况下控制面板只能在本地访问（localhost），请勿将 18789 端口直接暴露在公网，否则任何人都可能访问你的 AI 助手并控制你的电脑。

3

✋ 开启"同意模式"

建议新手在配置文件中开启 exec.ask: "on" 选项，这样 AI 在执行任何写入或命令操作前都会先请求你的确认，避免意外操作。

4

🔌 谨慎安装技能插件

只安装社区验证过的技能插件，不要随意安装来源不明的插件，它们可能包含恶意代码。

5

👤 为 AI 创建专用账户

不要直接让 AI 使用你的个人账户（如 Gmail、GitHub 等），建议创建专用的子账户或应用密码，并只授权必要的访问权限。

10

常见问题 FAQ

安装脚本运行后什么反应都没有，怎么办？ ▼
可能是网络问题。请检查网络连接，确认可以访问 openclaw.ai。如果在中国大陆，可能需要使用代理。另外检查终端是否有权限运行 curl 命令：which curl，如果没有输出，需要先安装 curl。
提示 "command not found: openclaw"？ ▼
这是 PATH 环境变量问题，npm 全局安装目录没有添加到系统路径。在 macOS/Linux 中运行：

echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

如果使用 bash，将 .zshrc 替换为 .bashrc。Windows 用户需要将 npm prefix -g 的输出路径添加到系统 PATH 环境变量中。
网关没有响应，或提示 "0 tokens used"？ ▼
这通常是 Daemon 停止运行或 API 密钥验证失败导致的。请依次尝试：

1. 运行 openclaw gateway restart 重启网关
2. 运行 openclaw doctor 自动诊断配置问题
3. 进入配置文件检查 API 密钥是否正确
提示 "RPC probe: failed" 端口错误？ ▼
端口 18789 被其他程序占用了。运行以下命令查找并终止占用进程：

sudo lsof -i :18789

找到对应进程的 PID 后，用 kill -9 [PID] 终止它，然后重新启动 OpenClaw。
没有 AI API 密钥，可以免费使用 OpenClaw 吗？ ▼
可以！安装 Ollama 后可以在本地运行开源 AI 模型（如 Llama 3、Mistral 等），完全免费且不依赖云端 API。不过本地模型需要你的电脑有足够的内存（建议 8GB 以上），性能相比云端顶级模型有所差距。
如何卸载 OpenClaw？ ▼
运行以下命令：

npm uninstall -g openclaw

然后删除配置目录：rm -rf ~/.openclaw。如果安装了 Daemon，还需要分别在 Linux 运行 systemctl --user disable openclaw，或在 macOS 运行 launchctl unload ~/Library/LaunchAgents/openclaw.plist。
OpenClaw 会自动更新吗？ ▼
OpenClaw 不会自动更新，需要手动执行以下命令升级到最新版本：

npm update -g openclaw

更新后建议运行 openclaw doctor 检查配置是否需要迁移。
